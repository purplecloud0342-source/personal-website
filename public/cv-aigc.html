小黄人版老友记 — IP 混搭 AIGC 实验内容策略 + 全流程 AIGC 制作 2026

工具链选型：为什么跨三个平台而不是死磕一个？最初只用 Midjourney 出图 + 手动拼接，角色风格来回跳，同一角色上一张和下一张像两个人。试了 Runway 的图生视频后发现角色连贯性好不少，但表情僵硬。最后组合成 Midjourney 定角色形象 → Runway 做动态 → 可灵补表情细节的三段式管线。不是哪个工具最好，而是每个工具擅长的事不一样——Midjourney 角色设计强，Runway 动态连贯，可灵表情自然。跨平台不是炫技，是各自用长板

Prompt 调优与风格控制：早期每个片段单独写 Prompt，角色形象完全失控。后来设计了一套视觉锚点策略——给每个角色固定三个视觉特征（眼镜框型、发型轮廓、主配色），所有 Prompt 强制包含这三个锚点。可用素材率从 40% 拉到 75%，不是 Prompt 写得更好，而是把「角色一致性」这件事从技巧变成了规则。核心发现：AIGC 最大的敌人不是画质不够，是风格漂移——观众对连贯性的敏感度远超分辨率

失败与修正：前期做了 10 段素材后全部重来——IP 混搭的「度」没找对，太像原版没混搭效果，太不像又丢了 IP 辨识度。锁定「70% 老友记场景 + 30% 小黄人特征」的配比后才稳定。另一个问题是工具链断裂——MJ 出的图直接丢给 Runway 效果很差，中间加了 Topaz 放大和手动修图一步，质量才够用

成果：小红书 9,863 播放、1,564 赞。验证了核心判断：有传播力的创意组合 + 够用的制作质量，胜过大制作但缺少传播基因的炫技

AI 故事板 — 多 Agent 剧本与分镜生成系统Agent 系统设计 + 全栈开发 2026

架构选型：为什么是多 Agent 而不是一个端到端生成？测过直接让 LLM 从想法出分镜——出来的东西像大纲，没有镜头语言。拆成三个 Agent 后质量完全不同：编剧 Agent 出剧本结构 → 分镜 Agent 把每场戏翻译成镜头描述（景别、角度、运动）→ 视觉 Agent 生成参考画面。每个 Agent 只对上一个的输出负责，下游出问题不用改上游。拆开的本质好处：每个环节可独立评测，剧本不行就只调编剧 Prompt，不用推翻整个管线

Prompt 调优：分镜 Agent 第一版 Prompt 要求「把剧本转成分镜」，输出全是远景固定机位，没有任何导演思维。迭代后明确要求「至少有三种景别变化」「标注每个镜头的情绪功能（铺垫/推进/强调）」，输出才从机械翻译变成有叙事意图的镜头设计。

失败与修正：最早想把分镜图直接用 AI 出成成品级画面，结果风格跨场景不统一，上一个镜头日系下一个镜头写实，根本接不起来。后来降低预期——画面只做到「参考级」而非「成品级」，统一用线稿或低精度渲染，确保风格一致。核心教训：AIGC 内容的「可看度」比单帧「精美度」重要得多，一张超精美的图如果和前后镜头断裂，对成片破坏远大于一张合格的统一风格图

成果：从想法到带画面的分镜剧本的端到端管线跑通。该经历对创作流程拆解和 AI 管线编排的理解，可直接迁移至 AIGC 内容生产流程设计

三七互娱 — 微电影后期剪辑 + 调色 2026

叙事策略：为什么叠化而不是硬切？这部微电影讲两个人跨越不同时空的关联。硬切强调对比，叠化强调联系——这个故事的核心是「不同时空的同一种感受」，所以用叠化构建两个时空之间的视觉桥梁，观众不会觉得场景切换是断裂的。剪辑方法不是套路，是服务于叙事目的

调色决策：前期素材色温和曝光不一致，不同场景拍得像不同项目。先统一色温到暖调基准（5400K），再逐个镜头恢复肤色——肤色是观众注意力停留最久的区域，肤色不准整部片感觉不对。然后用光影对比强化面部——关键情绪点提亮面部、压暗背景，引导观众注意力。全程 DaVinci Resolve，从素材到成片独立交付
